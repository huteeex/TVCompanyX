## Шаг 1 — Подготовить базу данных

Открыть PowerShell и проверить, что БД существует и заполнена:

```powershell
$env:PGPASSWORD = "0112"
psql -U postgres -h localhost -d TVVVV -c "\dt"
```

Должны появиться таблицы:
`users`, `show_types`, `tv_show`, `show_schedule`, `request_status_type`,
`payment_type`, `request`, `contract`, `notification_type`, `notification`,
`chat_message`, `audit_log`

Если таблицы отсутствуют — выполнить скрипт инициализации:

```powershell
$env:PGPASSWORD = "0112"
psql -U postgres -h localhost -d TVVVV -f "l:\project\TVCompanyX\docs\db\init_full_schema.sql"
```

Проверить наличие пользователей:

```powershell
psql -U postgres -h localhost -d TVVVV -c "SELECT user_id, first_name, last_name, email, role FROM users LIMIT 5;"
```

Убедиться, что хотя бы один пользователь существует. Если нет — вставить тестового:

```sql
INSERT INTO users (first_name, last_name, email, password_hash, role, is_active, updated_at)
VALUES ('Иван', 'Тест', 'admin@test.ru',
  encode(sha256('password123'::bytea), 'hex'),
  'admin', true, now());
```

### ЛР-12.2 — Безопасность паролей (SHA-256)

Пароли хранятся в таблице `users.password_hash` как SHA-256 hex-строки (64 символа).

Проверить:
```powershell
psql -U postgres -h localhost -d TVVVV -c "SELECT email, LENGTH(password_hash), password_hash FROM users LIMIT 3;"
```

Ожидаемый результат: `length = 64`, строка состоит только из hex-символов `[0-9a-f]`.

## Быстрый сценарий полного теста (end-to-end)

Выполнить последовательно:

1. **Запустить** `dotnet run --urls http://localhost:5050`
2. **Открыть** http://localhost:5050 → попасть на `/Account/Login`
3. **Войти** с данными тестового пользователя  
4. На главной — проверить дашборд: счётчики телешоу, заявок, пользователей
5. `/ShowTypes` → создать тип «Реклама»
6. `/TvShows` → создать шоу «Утренний эфир», тип «Реклама», цена 500 руб/мин
7. `/ShowSchedules` → создать запись: шоу «Утренний эфир», дата завтра, время 09:00
8. `/PaymentTypes` → создать «Безналичный расчёт»
9. `/RequestStatusTypes` → создать «Новая»
10. `/Requests` → создать заявку: заказчик = текущий пользователь, расписание = только что созданное
11. `/Contracts` → создать договор к заявке
12. **Выйти** → убедиться, что страницы недоступны без авторизации

## Возможные проблемы

### Ошибка подключения к БД при запуске

```
Unable to connect to database
```

**Решение:** проверить, что PostgreSQL запущен:
```powershell
Get-Service -Name "postgresql*"
# Если stopped:
Start-Service -Name "postgresql-x64-18"
```

### Страница 500 при открытии списка

Скорее всего, таблица-справочник пуста и выпадающий список не загружается.  
**Решение:** заполнить справочники перед созданием записей в основных таблицах (порядок: ShowTypes → TvShows → ShowSchedules → Requests).

### Файл занят при `dotnet build`

```
error MSB3027: файл занят другим процессом
```

**Решение:**
```powershell
Stop-Process -Name "Lab12TVCompanyX" -Force -ErrorAction SilentlyContinue
dotnet run --urls "http://localhost:5050"
```

### Пароль не принимается при входе

Убедиться, что пароль в БД хранится как **SHA-256 hex** (64 символа):
```powershell
psql -U postgres -h localhost -d TVVVV -c "SELECT email, LENGTH(password_hash) FROM users;"
```
Если `length ≠ 64` — пользователь создан со старым хешем или plain-text. Пересоздать через интерфейс `/Users`.
