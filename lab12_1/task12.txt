Краткие теоретические сведения
ASP.NET Core — это современный, кроссплатформенный (Windows, macOS, Linux) и с открытым исходным кодом фреймворк от Microsoft для создания высокопроизводительных веб-приложений, веб-сервисов и мобильных бэкендов, работающих на платформе .NET. Он отличается модульностью, гибким развертыванием (включая Docker) и высокой производительностью, являясь эволюцией классического ASP.NET. 
Основные характеристики:
Кроссплатформенность: Разработка и запуск на Windows, macOS, Linux и в контейнерах.
Открытый исходный код: Разрабатывается Microsoft и сообществом.
Модульность: Минимальная загрузка, возможность выбирать нужные компоненты.
Производительность: Оптимизирован для современных облачных сценариев и показал высокие результаты в тестах.
Гибкое развертывание: Приложение можно развернуть вместе со средой выполнения или установить централизованно.
Поддержка языков: Основные языки — C# и F#.
Современные подходы: Включает внедрение зависимостей (DI) и промежуточное ПО (middleware).
Entity Framework Core (EF Core) — это современный объектно-реляционный модуль сопоставления (ORM) для .NET, который позволяет разработчикам работать с базами данных с помощью объектов .NET (классов C#) вместо написания SQL-запросов, упрощая доступ к данным, отслеживание изменений, миграции схемы и поддержку различных СУБД, таких как SQL Server, MySQL, PostgreSQL и SQLite. 
Основные возможности и преимущества EF Core:
ORM (Object-Relational Mapping): Сопоставляет таблицы базы данных с объектами .NET, позволяя манипулировать данными через объекты и коллекции, а не строки и столбцы.
Поддержка LINQ: Позволяет писать запросы к базе данных, используя синтаксис Language Integrated Query (LINQ) прямо в C#, что делает код более читаемым.
Кроссплатформенность: Работает на Windows, Linux и macOS, что делает его идеальным для современных приложений .NET.
Расширяемость: Позволяет добавлять провайдеры для разных баз данных (SQL Server, SQLite, MySQL, PostgreSQL, Azure Cosmos DB и др.).
Миграции: Управляет изменениями схемы базы данных, позволяя легко обновлять структуру БД в соответствии с изменениями в моделях .NET.
Устранение бойлерплейт-кода: Значительно сокращает объем шаблонного кода, необходимого для доступа к данным.
MVC (Model-View-Controller) — это архитектурный паттерн проектирования приложений, который разделяет их на три взаимосвязанные части: Модель (данные и логика), Представление (пользовательский интерфейс) и Контроллер (посредник, обрабатывающий действия пользователя и связывающий Модель с Представлением), что позволяет разработчикам управлять сложностью кода, делая его более модульным и легким для поддержки. 
Компоненты MVC:
Модель (Model): Управляет данными и бизнес-логикой приложения. Она не знает о том, как данные будут отображаться, и реагирует на запросы контроллера, изменяя свое состояние или предоставляя данные.
Представление (View): Отвечает за визуализацию данных из Модели для пользователя. Это то, что видит пользователь (интерфейс). Одно и то же представление может использовать разные Модели, и наоборот.
Контроллер (Controller): Принимает ввод от пользователя (например, нажатие кнопки), обрабатывает его и взаимодействует с Моделью для изменения данных. Затем он выбирает соответствующее Представление для отображения результата.

using Example_Project.Models;
/******* Начало модификации *******/
using Microsoft.AspNetCore.Authentication.Cookies;
using Microsoft.EntityFrameworkCore;
/******* Конец модификации *******/
namespace Example_Project
{
    public class Program
    {
        public static void Main(string[] args)
        {
            var builder = WebApplication.CreateBuilder(args);

            // Add services to the container.
            builder.Services.AddControllersWithViews();

            /******* Начало модификации *******/
            // Добавляем контекст БД
            builder.Services.AddDbContext<MyDbContext>(options =>
                options.UseSqlServer(builder.Configuration.GetConnectionString("DefaultConnection")));

            // Настраиваем аутентификацию через куки
            builder.Services.AddAuthentication(CookieAuthenticationDefaults.AuthenticationScheme)
                .AddCookie(options =>
                {
                    options.LoginPath = "/Account/Login";
                    options.AccessDeniedPath = "/Account/AccessDenied";
                });
            /******* Конец модификации *******/


            var app = builder.Build();

            // Configure the HTTP request pipeline.
            if (!app.Environment.IsDevelopment())
            {
                app.UseExceptionHandler("/Home/Error");
                // The default HSTS value is 30 days. You may want to change this for production scenarios, see https://aka.ms/aspnetcore-hsts.
                app.UseHsts();
            }

            app.UseHttpsRedirection();
            app.UseStaticFiles();

            app.UseRouting();
            
            /******* Начало модификации *******/
            // ВАЖНО: UseAuthentication ДО UseAuthorization
            app.UseAuthentication();
            /******* Конец модификации *******/


            app.UseAuthorization();

            app.MapControllerRoute(
                name: "default",
                pattern: "{controller=Home}/{action=Index}/{id?}");

            app.Run();
        }
    }
}

using Example_Project.Models;
using Microsoft.AspNetCore.Authentication;
using Microsoft.AspNetCore.Authentication.Cookies;
using Microsoft.AspNetCore.Mvc;
using Microsoft.EntityFrameworkCore;
using System.Security.Claims;

namespace MyWebApp.Controllers
{
    public class AccountController : Controller
    {
        private readonly MyDbContext _context;

        public AccountController(MyDbContext context)
        {
            _context = context;
        }
/******* Конец модификации *******/

        public IActionResult Login()
        {
            return View();
        }


/******* Начало модификации *******/

        [HttpPost]
        public async Task<IActionResult> Login(string login, string password)
        {
            var user = await _context.Users
                .FirstOrDefaultAsync(u => u.Login == login && u.Password == password);

            if (user != null)
            {
                var claims = new List<Claim>
                {
                    new Claim(ClaimTypes.Name, user.Login),
                    new Claim("UserID", user.UserId.ToString())
                };

                var claimsIdentity = new ClaimsIdentity(
                    claims, CookieAuthenticationDefaults.AuthenticationScheme);

                await HttpContext.SignInAsync(
                    CookieAuthenticationDefaults.AuthenticationScheme,
                    new ClaimsPrincipal(claimsIdentity));

                return RedirectToAction("Index", "Home");
            }

            ViewBag.Error = "Неверный логин или пароль";
            return View();
        }

        public async Task<IActionResult> Logout()
        {
            await HttpContext.SignOutAsync(CookieAuthenticationDefaults.AuthenticationScheme);
            return RedirectToAction("Login", "Account");
        }
    }
}
/******* Конец модификации *******/

using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.Mvc;
using System.Diagnostics;

namespace Example_Project.Controllers
{
    [Authorize] // Добавляем этот атрибут
    public class HomeController : Controller
    {
        private readonly ILogger<HomeController> _logger;

        public HomeController(ILogger<HomeController> logger)
        {
            _logger = logger;
        }

        public IActionResult Index()
        {
            ViewBag.UserName = User.Identity.Name; // Получаем логин пользователя
            return View();
        }

        public IActionResult Privacy()
        {
            return View();
        }

        [ResponseCache(Duration = 0, Location = ResponseCacheLocation.None, NoStore = true)]
        public IActionResult Error()
        {
            return View(new ErrorViewModel { RequestId = Activity.Current?.Id ?? HttpContext.TraceIdentifier });
        }
    }
}
